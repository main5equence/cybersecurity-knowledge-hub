# Network Protocols

## Overview

Network protocols are sets of rules that define how data is transmitted and received across a network.

They ensure reliable communication between devices.

---

## Common Protocols

### HTTP / HTTPS
- Used for web communication  
- HTTPS provides encryption  

### TCP (Transmission Control Protocol)
- Reliable, connection-based communication  
- Ensures data is delivered correctly  

### UDP (User Datagram Protocol)
- Faster but less reliable  
- Used in streaming and gaming  

### IP (Internet Protocol)
- Responsible for addressing and routing  

### DNS (Domain Name System)
- Translates domain names into IP addresses  

---

## Secure Protocols

- HTTPS (secure HTTP)  
- SSH (secure remote access)  
- TLS/SSL (encryption protocols)  

---

## Risks and Weaknesses

- Unencrypted communication  
- Protocol misconfiguration  
- DNS spoofing  

---

## Key Takeaways

- Protocols define how devices communicate  
- Some protocols are secure, others require additional protection  
- Understanding protocols helps identify vulnerabilities  

---

## Notes

Secure communication depends on choosing the right protocols and configuring them properly.
