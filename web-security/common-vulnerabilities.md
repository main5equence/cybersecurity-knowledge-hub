# Common Web Vulnerabilities

## Overview

Web vulnerabilities are weaknesses in web applications that can be exploited by attackers.

They often result from poor design, misconfiguration, or insecure coding practices.

---

## Common Vulnerabilities

### SQL Injection
- Allows attackers to manipulate database queries.

### Cross-Site Scripting (XSS)
- Injects malicious scripts into web pages.

### Cross-Site Request Forgery (CSRF)
- Forces users to perform unintended actions.

### Broken Authentication
- Improper implementation of authentication mechanisms.

### Security Misconfiguration
- Incorrect system or application setup.

---

## Causes

- Lack of input validation  
- Poor coding practices  
- Outdated software  
- Misconfigured systems  

---

## Prevention Methods

- Validate and sanitize input  
- Use secure frameworks  
- Keep systems updated  
- Apply proper access controls  

---

## Key Takeaways

- Vulnerabilities are often preventable  
- Secure coding and configuration are essential  
- Regular testing helps identify weaknesses  

---

## Notes

Understanding common vulnerabilities is key to building and maintaining secure web applications.
