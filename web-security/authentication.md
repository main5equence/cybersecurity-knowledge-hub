# Authentication

## Overview

Authentication is the process of verifying the identity of a user or system.

It ensures that only authorized individuals can access specific resources.

---

## Authentication Factors

Authentication is typically based on one or more of the following:

- **Something you know** (password, PIN)  
- **Something you have** (token, smartphone)  
- **Something you are** (biometrics)  

---

## Multi-Factor Authentication (MFA)

MFA combines multiple authentication factors to increase security.

### Example:
- Password + SMS code  

### Benefit:
- Reduces risk of unauthorized access  

---

## Common Issues

- Weak passwords  
- Reuse of credentials  
- Lack of MFA  
- Poor session management  

---

## Best Practices

- Use strong, unique passwords  
- Enable MFA  
- Implement secure session handling  
- Monitor login activity  

---

## Key Takeaways

- Authentication verifies identity  
- Multiple factors increase security  
- Weak authentication is a major vulnerability  

---

## Notes

Authentication is a critical component of access control and system security.
